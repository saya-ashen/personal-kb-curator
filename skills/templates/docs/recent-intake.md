# Recent Intake

Use this file to stage recent candidate items before full canonical merge.

For each candidate, record:

- title
- source
- date
- topic
- why it matters
- suggested target path
- status (`candidate`, `approved`, `merged`, `rejected`, `duplicate`)
- next action

## Token-economy defaults

- keep only active items in this file (`candidate`, `approved`)
- cap active rows at 50 by default
- move resolved rows (`merged`, `rejected`, `duplicate`) to monthly archive
  files, for example `archive/recent-intake-YYYY-MM.md`
- keep each row short and structured; avoid long narrative notes
